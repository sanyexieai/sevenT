# Chrome 插件安装与配置指南

这份文档面向普通使用者，目标是用最少步骤把扩展装好并跑起来。

## 1. 先准备翻译服务

扩展本身只负责抓取网页文本和把译文写回页面，真正的翻译由 HTTP 服务完成。

你可以任选一种方式：

- 使用本仓库发布页里的后端构建包
- 自己在本地运行 `cargo run`
- 使用已经部署好的兼容接口

如果你使用本仓库自带的 NLLB 后端，还需要提前准备模型目录，并放到仓库里的这个位置：

```text
models/nllb-200-distilled-600M/
```

如果还没有准备模型，先看 README 里的“Model Preparation / 模型准备”章节，再继续安装扩展。

如果你使用本仓库后端，默认接口地址是：

```text
http://127.0.0.1:8080
```

建议先验证服务是否正常：

```bash
curl http://127.0.0.1:8080/healthz
```

看到可用响应后，再继续安装扩展。

## 2. 下载扩展发布包

从 GitHub Releases 下载扩展发布包。常见有两种：

```text
nas-page-translator-chrome-extension-v*.zip
nas-page-translator-chrome-extension-v*.crx
```

推荐优先使用 `.crx`。如果浏览器不允许本地 `.crx` 安装，再改用 `.zip` 解压安装。

### 方案 A：安装 `.crx`

1. 打开 `chrome://extensions`
2. 打开右上角“开发者模式”
3. 下载 `.crx` 后，如果 Chrome 下载栏或下载页面提示该文件不常见、不安全或可能有风险，先选择“保留”或“仍然保留”
4. 把下载好的 `.crx` 文件拖到扩展页
5. 在弹出的确认框中点击“添加扩展程序”完成安装

如果浏览器先弹出额外安全确认，按提示允许本地安装即可。
不同版本的 Chrome 文案可能略有不同，但核心动作通常是“保留文件”加“确认安装”。
安装完成后，扩展页里还可能出现“安全检查”“Chrome 无法验证此扩展程序的来源”之类的提示。这是本地分发 `.crx` 的常见现象，不代表安装失败。

如果浏览器拒绝安装，再使用下面的 `.zip` 解压安装方式。

### 方案 B：安装 `.zip`

下载后把压缩包解压到一个固定目录，例如：

```text
Downloads/nas-page-translator/
```

不要把它放在会被系统自动清理的临时目录里，也不要解压后再随意移动目录。

## 3. 在 Chrome 里安装

如果你使用的是 `.zip`，按下面步骤安装：

1. 打开 `chrome://extensions`
2. 打开右上角“开发者模式”
3. 点击“加载已解压的扩展程序”
4. 选择刚才解压出来的扩展目录
5. 确认扩展已经出现在列表中

安装完成后，建议把扩展固定到工具栏，方便后续配置和使用。

## 4. 第一次配置

点击工具栏中的扩展图标，按下面方式填写：

- `Provider`
  - 使用本仓库后端时，选本地 NLLB HTTP 服务对应的 provider
  - 使用 OpenAI 兼容接口时，选 OpenAI 兼容 provider
- `Endpoint`
  - 本地后端默认填 `http://127.0.0.1:8080`
- `Render Mode`
  - `compare`：保留原文，在附近追加译文
  - `replace`：直接把页面文字替换成译文
- `Source Lang`
  - 一般填 `eng_Latn`
- `Target Lang`
  - 一般填 `zho_Hans`
- `Max Items`
  - 普通网页可以先用默认值，页面特别复杂时再逐步调大

如果你使用 OpenAI 兼容接口，还需要补充：

- `Model`
- `API Key`

填完后点击“保存配置”即可。
如果直接点击“翻译整页”，插件也会先自动保存当前修改，再执行翻译。

## 5. 开始使用

打开任意普通网页后，可以通过下面任一方式触发：

- 点击扩展弹窗中的整页翻译按钮
- 点击页面右侧的悬浮按钮
- 在页面空白处右键，使用“翻译整页为中文”

常用模式说明：

- 想对照原文阅读，用 `compare`
- 想把页面整体换成中文，用 `replace`

## 6. 常见问题

### 点击扩展没有反应

优先检查：

- 当前页面是不是 `chrome://`、扩展页、新标签页之类的浏览器内置页面
- `http://127.0.0.1:8080/healthz` 是否能访问
- 扩展是否在最近更新后还没重新加载

### 已经装好了，但翻译失败

优先检查：

- `Endpoint` 地址是否填错
- 后端是否真的启动成功
- 模型目录是否完整
- OpenAI 兼容接口的 `Model` 和 `API Key` 是否正确

### 翻页后没有继续翻译

先确认：

- 之前已经成功执行过一次“整页翻译”
- 扩展已经更新到最新版本
- 该网页不是浏览器完全禁止注入脚本的页面

### 能不能直接双击 zip 安装

不能。当前分发方式是“下载发布包后解压，再通过 `chrome://extensions` 加载已解压扩展”。

如果你拿到的是 `.crx`，可以优先尝试拖到扩展页安装。
如果浏览器拦截本地 `.crx`，就退回到 `.zip` 解压安装。

### 下载 `.crx` 时提示有风险怎么办

这类提示通常是因为 `.crx` 不是从 Chrome Web Store 下载的，而不是因为扩展一定有问题。

常见处理方式：

1. 在下载栏、下载气泡或下载页面里找到这次 `.crx` 下载
2. 选择“保留”“仍然保留”或类似选项
3. 再把 `.crx` 拖到 `chrome://extensions`
4. 在安装确认框里继续完成安装

如果你的 Chrome 不允许继续本地 `.crx` 安装，就直接改用 `.zip` 解压安装，这也是最稳的方式。

### 安装后提示“无法验证来源”怎么办

如果扩展已经成功安装，但在 `chrome://extensions` 里看到类似下面的提示：

- “安全检查”
- “Chrome 建议您移除它”
- “Chrome 无法验证此扩展程序的来源”

这通常是因为扩展不是从 Chrome Web Store 安装，而是本地手动安装的。

可以这样判断是否属于正常现象：

- 扩展已经出现在扩展列表中
- 状态显示为“已启用”
- 你可以正常打开扩展弹窗
- 扩展在网页上可以正常工作

如果以上都正常，这类提示一般可以视为本地分发带来的来源提醒。
如果你更希望没有这类来源提示，需要把扩展正式发布到 Chrome Web Store。
